# BOB-perf 사용법

## 사전 준비

```bash
# 1. 모니터링 환경 실행
docker-compose up -d

# 2. 실행 확인
docker ps
# prometheus, grafana 컨테이너가 running 상태인지 확인
```

---

## 1. 서버 상태 모니터링

### Grafana 접속

1. 브라우저에서 http://localhost:3000 접속
2. 로그인: `admin` / `admin`

### 대시보드 선택

| 대시보드 | 용도                               |
|---------|----------------------------------|
| **Spring Boot - Local** | 로컬 서버 모니터링 (histogram 기반, 상세 메트릭) |
| **Spring Boot - Production** | EC2 운영 서버 모니터링                   |
| **Spring Boot - Development** | EC2 개발 서버 모니터링                   |
| **k6 Load Test Results** | k6 부하 테스트 결과 확인                  |

### 확인 가능한 메트릭

- TPS (초당 요청 수)
- Response Time (P50, P95, P99)
- Error Rate
- JVM Heap Memory
- CPU Usage
- HikariCP Connection Pool

## 2. 부하 테스트 실행 및 결과 확인

### 테스트 실행

```powershell
# Windows
.\k6\scripts\run-k6-test.ps1 [옵션]

# Mac/Linux
./k6/scripts/run-k6-test.sh [옵션]
```

### 파라미터 옵션

| 파라미터 | 설명 | 값 | 기본값 |
|---------|------|-----|--------|
| `-Scenario` / `-s` | 테스트 시나리오 | baseline, load, stress | load |
| `-Target` / `-t` | 테스트 대상 서버 | local, dev | dev |
| `-MaxVUs` / `-v` | 최대 가상 사용자 수 | 숫자 | 시나리오별 기본값 |
| `-TestFile` / `-f` | 테스트 스크립트 | 파일명 (.js 제외) | post-list-test |

### 시나리오별 기본 VU

| 시나리오 | 설명 | 기본 VU |
|---------|------|---------|
| `baseline` | 기본 동작 확인 | 1 VU, 1분 |
| `load` | 예상 트래픽 테스트 | 300 VU, 3분 |
| `stress` | 한계점 탐색 | 100→500 VU, 3분 단계적 증가 |

### 실행 예시

```powershell
# Windows

# 베이스라인 테스트 (서버 정상 동작 확인)
.\k6\scripts\run-k6-test.ps1 -Scenario baseline

# 부하 테스트 (기본값: dev 서버, 300 VU)
.\k6\scripts\run-k6-test.ps1 -Scenario load

# 로컬 서버 대상 부하 테스트
.\k6\scripts\run-k6-test.ps1 -Scenario load -Target local

# VU 수 커스텀 (500 VU)
.\k6\scripts\run-k6-test.ps1 -Scenario load -MaxVUs 500

# 다른 테스트 스크립트 실행
.\k6\scripts\run-k6-test.ps1 -TestFile post-detail-test -Scenario load

# 복합 옵션
.\k6\scripts\run-k6-test.ps1 -Scenario stress -Target dev -MaxVUs 1000 -TestFile post-list-test
```

```bash
# Mac/Linux

# 부하 테스트 (기본값)
./k6/scripts/run-k6-test.sh -s load

# 로컬 서버 대상, 200 VU
./k6/scripts/run-k6-test.sh -s load -t local -v 200

# 다른 테스트 스크립트 실행
./k6/scripts/run-k6-test.sh -f post-detail-test -s load
```

```powershell
# Windows

# 부하 테스트 (기본값)
.\k6\scripts\run-k6-test.ps1 -Scenario load

# 로컬 서버 대상, 200 VU
.\k6\scripts\run-k6-test.ps1 -Scenario load -Target local -MaxVUs 200

# 다른 테스트 스크립트 실행
.\k6\scripts\run-k6-test.ps1 -TestFile post-detail-test -Scenario load
```

### 결과 확인

**1. 터미널 출력**
- 테스트 종료 후 요약 정보 출력
- Total Requests, Avg Duration, P95, RPS 등

**2. Grafana 실시간 모니터링**
1. http://localhost:3000 접속
2. **k6 Load Test Results** 대시보드 선택
3. 확인 가능한 메트릭:
   - Active VUs (가상 사용자 수)
   - Requests/sec (RPS)
   - Response Time (P50, P90, P95, P99)
   - Error Rate (%)
   - HTTP Status Codes

**3. 서버 상태와 비교 분석**
- **Spring Boot** 대시보드와 **k6** 대시보드를 함께 보면서
- "VU 300 넘어가니까 CPU 80%, 응답시간 급증" 같은 상관관계 파악

## 테스트 대상 서버

| Target | URL |
|--------|-----|
| `local` | http://localhost:8080 |
| `dev` | https://dev.bookbridge.kr |

```powershell
# -Target 파라미터로 간편하게 선택
.\k6\scripts\run-k6-test.ps1 -Target local   # 로컬 서버
.\k6\scripts\run-k6-test.ps1 -Target dev     # 개발 서버 (기본값)
```

---

## 테스트 스크립트 추가

`k6/tests/` 폴더에 새로운 테스트 파일을 추가한 후 `-TestFile` 파라미터로 실행:

```powershell
# k6/tests/post-detail-test.js 파일 생성 후
.\k6\scripts\run-k6-test.ps1 -TestFile post-detail-test -Scenario load
```
